■ 한글 일부만 보이고 나머지가 □로 나오는 현상
  원인: HeirofLightBold SDF_2 에셋에 한글 글리프가 일부만 포함되어 있음.
  (뒤, 로, 보, 상, 미, 기 등은 포함, 시·나·리·오·진·행·편·투·자·동·성 등은 미포함)

■ 해결 방법 (Unity 에디터에서 수행)

1. 메뉴: Window > TextMesh Pro > Font Asset Creator

2. 설정
   - Source Font File: Assets/TextMesh Pro/Fonts/HeirofLightBold.ttf
   - Character Set: Unicode Range (Hex)
   - Character Sequence (Hex): 20-7E,AC00-D7A3
     (ASCII 20-7E + 한글 음절 전체 AC00-D7A3)
   - Atlas Resolution: 한글 전체 포함 시 2048×2048 또는 4096×4096 권장
     (1024로 하면 글리프가 잘릴 수 있음)
   - Render Mode: SDFAA 등 기존과 동일하게

3. [Generate Font Atlas] 실행 후, 기존 에셋 덮어쓰기
   - Save As: Assets/Project/Fonts/HeirofLightBold SDF_2.asset
   (또는 새 에셋으로 저장 후, LobbyView.uss / MapView.uss / BattleReadyStyles.uss
    안의 -unity-font-definition url을 새 에셋 경로로 변경)

4. UI Toolkit에서 사용 중인 폰트가 TMP SDF 에셋이면, 일부 환경에서는
   TextCore Font Asset을 별도 생성해 쓰는 것이 안정적일 수 있음.
   (Create > Text > Font Asset - 같은 ttf로 한글 범위 포함해 생성 후
    Panel Settings 또는 USS에서 해당 에셋 참조)
